CSS Media Queries & Breakpoints

1. The Media Query

Media queries use the @media rule to apply block styles based on specific hardware features.

  • Media Types: Target devices directly using keywords like screen or print.
  • Media Features: Query conditions such as width, orientation, or resolution.
  • Operators: Combine multiple logic rules using and, not, and only.

2. Choosing Breakpoints

Breakpoints are the screen width thresholds where the layout transitions to fit content comfortably.

  • Mobile Threshold: Under 768px. Optimizes single column layouts for thumbs.
  • Tablet Range: Between 768px and 1024px. Typically handles split column grids.
  • Desktop Base: Over 1024px. Utilizes wide multi-column navigation layouts.

3. Mobile-First Approach

Modern responsive engineering structures styles to prioritize performance on mobile screens first.

  • Mobile First: CSS is written cleanly for mobile first; then min-width queries add desktop layout extensions.
  • Desktop First: Desktop layout defaults; then max-width overrides scale elements down.
  • Benefits: Reduces browser computation and ensures page load performance.

@media (min-width: 1025px) {
  /* Desktop Styles */
  body {
    background-color: #f8fafc;
  }
  .cards-container {
    flex-wrap: nowrap;
  }
}
@media (max-width: 1024px) {
  /* Tablet Styles */
  body {
    background-color: #f5f3ff;
  }
  .card {
    flex: 1 1 calc(50% - 1.5rem);
  }
}
@media (max-width: 768px) {
  /* Mobile Styles */
  body {
    background-color: #fdf2f8;
  }
  .cards-container {
    flex-direction: column;
  }
}
